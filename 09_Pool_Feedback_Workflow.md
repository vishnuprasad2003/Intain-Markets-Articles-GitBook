# Pool Feedback Workflow

## Overview

The feedback workflow allows market makers, investors, and other parties to provide input on pools shared with them. This collaborative process helps improve pool quality and ensures all parties are aligned before finalizing deals.

## Who Can Use This

- Market Makers who review pools
- Investors who review opportunities
- Rating Agencies who analyze pools
- Issuers who receive and respond to feedback

## When This Is Used

Use feedback workflow when:
- Reviewers want to request changes or provide comments
- Issuers need to address feedback and improve pools
- Parties want to discuss pool details
- Change requests need to be communicated

## Step-by-Step Process

### Providing Feedback

1. **Review the Pool**
   - View pool details and metrics
   - Analyze loan characteristics
   - Identify areas needing attention

2. **Decide Feedback Type**
   - General feedback: Comments or questions
   - Change request: Formal request for modifications
   - Choose pool-level or loan-level feedback

3. **Submit Feedback**
   - Enter your feedback message
   - Specify if it's a change request
   - Submit the feedback
   - Issuer receives notification

### Responding to Feedback

1. **Review All Feedback**
   - Check change requests carefully
   - Read comments and questions
   - Note any flagged items

2. **Prioritize Actions**
   - Address critical issues first
   - Handle blocking problems
   - Then address other feedback

3. **Make Changes**
   - Update pool information
   - Fix identified issues
   - Address comments
   - Resolve flagged items

4. **Communicate**
   - Respond to comments
   - Explain changes made
   - Ask questions if unclear
   - Confirm completion

5. **Resubmit**
   - When changes are complete
   - Resubmit for review
   - Status updates accordingly

## Rules & Validations

- You can only provide feedback if sharing permissions allow it
- Change requests require issuers to make changes and resubmit
- Feedback can be pool-level or loan-level
- Unread feedback is tracked and highlighted
- Issuers can respond to feedback by making changes or adding comments

## What Happens Next

After providing feedback:
- Issuers receive notifications about feedback
- Change requests require pool updates and resubmission
- General feedback can be addressed through discussion
- Feedback history is maintained for audit purposes
- Pools improve through iterative feedback and changes

The feedback workflow enables effective collaboration, helping improve pool quality through constructive input and ensuring all parties are aligned before finalizing deals.
