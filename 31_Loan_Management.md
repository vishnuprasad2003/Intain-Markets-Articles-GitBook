# Loan Management

## Overview

Loan management covers all the tasks issuers perform to manage loans in the platform, including uploading loans, mapping them to pools, updating loan information, managing loan statuses, and handling loan removals and reinstatements.

## Uploading Loans

### Step 1: Prepare Loan Data

**Data preparation:**
- Organize loan data in Excel or CSV format
- Ensure all required fields included
- Verify data accuracy
- Check data format matches requirements

### Step 2: Upload Loans

**Upload process:**
1. Navigate to Loans section
2. Click "Upload Loans" or "Import Loans"
3. Select your loan data file
4. Upload file to system
5. System processes and standardizes data

### Step 3: Review Standardized Loans

**After upload:**
- System standardizes loan data
- Loans appear in "Unmapped" status
- Review standardized data
- Verify accuracy
- Ready for mapping

---

## Mapping Loans to Pools

### Step 1: Select Loans

**How to select:**
1. Navigate to loan list
2. Filter to show unmapped loans (if needed)
3. Select loans to map
4. Can select multiple loans at once

### Step 2: Choose Target Pool

**Select pool:**
1. Choose pool from dropdown
2. Verify correct pool selected
3. Confirm pool is appropriate
4. Ready to map

### Step 3: Confirm Mapping

**Process:**
1. Review selected loans
2. Verify target pool
3. Click "Map" or "Add to Pool"
4. Loans mapped to pool
5. Pool metrics update automatically

---

## Managing Loan Statuses

### Viewing Loan Status

**Where to see status:**
- Loan list shows current status
- Loan details page
- Pool's loan list
- Status badges and indicators

### Understanding Statuses

**Mapping status:**
- Unmapped: Not in any pool
- Mapped or Pool Name: In a specific pool

**Workflow status:**
- STANDARDISED: Processed and ready
- SUBMITTED: In a batch
- VERIFIED: Verified and validated

**Pool status:**
- Normal: Active in pool
- Reconsider: Needs review
- Removed: Excluded from pool
- Reinstated: Put back in pool

## Updating Loan Information

### When You Can Edit

**Can edit when:**
- Loan is unmapped
- Loan is in Draft status pools
- Loan status allows editing
- You have permission

### How to Update

**Process:**
1. Navigate to loan details
2. Click "Edit" button
3. Update information
4. Save changes
5. Updates reflected immediately

### What Can Be Updated

**Editable fields:**
- Loan details
- Borrower information
- Financial terms
- Collateral information
- Other loan attributes

## Managing Loan Status in Pools

### Marking for Reconsideration

**When to mark:**
- Loan has issues identified
- Loan needs review
- Data quality concerns
- Compliance questions

**How to mark:**
1. Select loan(s) in pool
2. Click "Mark for Reconsideration"
3. Loan status: Reconsider
4. Review process begins

### Removing Loans

**When to remove:**
- Loan doesn't meet pool criteria
- Loan has permanent issues
- Pool requirements changed
- Need to exclude loan

**How to remove:**
1. Select loan(s) in pool
2. Click "Remove" button
3. Confirm removal
4. Loan status: Removed
5. Pool metrics update

### Reinstating Loans

**When to reinstate:**
- Loan issues resolved
- Mistaken removal
- Loan now meets criteria
- Want to include again

**How to reinstate:**
1. Select removed loan(s)
2. Click "Reinstate" button
3. Confirm reinstatement
4. Loan status: Normal
5. Pool metrics update

## Best Practices

### Loan Upload

1. **Prepare data carefully**: Ensure accuracy before upload
2. **Follow format**: Use required data format
3. **Verify after upload**: Check standardized data
4. **Fix issues early**: Address problems immediately

### Loan Mapping

1. **Map systematically**: Organize mapping process
2. **Verify mapping**: Check loans mapped correctly
3. **Review metrics**: Ensure pool metrics updated
4. **Complete mapping**: Map all intended loans

### Status Management

1. **Monitor statuses**: Track loan statuses regularly
2. **Address issues**: Handle reconsideration promptly
3. **Manage removals**: Remove loans appropriately
4. **Track changes**: Monitor status updates

## Common Scenarios

### Scenario 1: Standard Management

1. Upload loans → Unmapped
2. Map to pool → Mapped
3. Create batch → SUBMITTED
4. Verify → VERIFIED
5. Loans ready for deal

### Scenario 2: Status Changes

1. Loan in pool → Normal
2. Issue identified → Reconsider
3. Issue resolved → Normal
4. Or removed if needed → Removed

### Scenario 3: Correction Process

1. Loan has data error → Identify issue
2. Remove temporarily → Removed
3. Correct loan data → Update information
4. Reinstate loan → Normal
5. Loan back in pool

## Troubleshooting

### "Can't upload loans"
- Check file format matches requirements
- Verify required fields included
- Ensure file size within limits
- Check for format errors

### "Can't map loans"
- Verify loans are unmapped
- Check pool status allows mapping
- Ensure you have permission
- Verify pool exists

### "Can't update loan"
- Check loan status allows editing
- Verify you have permission
- Ensure loan not locked
- Check pool status
